🎯 Introducción

El Microservicio de Documentos es una plataforma integral diseñada para gestionar la documentación de transportistas de manera automatizada e inteligente. Utiliza inteligencia artificial para clasificar documentos, validar información y mantener el control de vigencias de manera profesional y eficiente.

Características Principales

🎯 Gestión Completa

Administración integral de documentos por entidad (choferes, camiones, acoplados, empresas)

🤖 IA Integrada

Clasificación automática con inteligencia artificial mediante integración con Flowise

📊 Sistema de Semáforos

Control visual de vigencias con códigos de color (verde/amarillo/rojo)

🔍 Búsqueda Avanzada

Consultas por múltiples criterios con filtros inteligentes

📋 Aprobación Manual

Sistema de revisión con preview de documentos y validación humana

📈 Dashboard en Tiempo Real

Métricas y estadísticas actualizadas constantemente

📁 Pestañas/Módulos del Sistema

👥 1. Clientes (/documentos/clientes)
Propósito: Gestión de empresas clientes que requieren documentación específica de transportistas.

Funcionalidades:

  • Crear clientes: Registro completo con razón social y CUIT
  • Configuración por defecto: Selección de cliente principal para asignaciones automáticas
  • Configuración de sistema:
    • Delay de verificación de faltantes (minutos)
    • Horizonte sin vencimiento (años)
  • Gestión de requisitos: Definir documentos requeridos por cliente
  • Administración: Activar/desactivar/eliminar clientes

Casos de Uso:

  • Registrar nuevos clientes con requisitos documentales específicos
  • Configurar plantillas de documentación personalizada
  • Establecer cliente por defecto para streamlining de procesos
🚛 2. Empresas Transportistas (/documentos/empresas-transportistas)
Propósito: Administración integral de empresas de transporte que operan con los equipos registrados.
  • Registro de empresas: Datos completos con información legal y de contacto
  • Gestión de contacto: Múltiples números de WhatsApp para notificaciones
  • Asociación con equipos: Vinculación empresa-vehículo para trazabilidad
  • Control de documentación: Seguimiento específico por empresa transportista
👤 3. Choferes (/documentos/choferes)
Propósito: Gestión completa de conductores y su documentación personal con validación automática.
  • Registro por DNI: Datos personales completos con validación
  • Múltiples teléfonos: Hasta 3 números WhatsApp con formato internacional
  • Búsqueda avanzada: Por DNI, nombre o apellido con filtros inteligentes
  • Paginación eficiente: Manejo optimizado de grandes volúmenes de datos
  • Gestión rápida: Activar/desactivar/eliminar con confirmación

Casos de Uso:

  • Registrar nuevos conductores con validación automática de datos
  • Actualizar información de contacto para sistema de notificaciones
  • Búsqueda rápida de choferes para asignación de equipos

Workflow de Aprobación:

  1. Lista de pendientes: Vista organizada de documentos esperando revisión
  2. Selección y preview: Click en documento para revisar con vista previa completa
  3. Verificación inteligente: Contraste entre datos detectados por IA y realidad
  4. Corrección asistida: Ajuste de campos con validación automática
  5. Decisión documentada: Aprobar o rechazar con motivo detallado
  6. Trazabilidad completa: Registro automático de decisión y auditoría

⚙️ Configuración del Sistema

🤖 Flowise (IA) - /configuracion/flowise
Propósito: Configuración de la inteligencia artificial para clasificación automática de documentos.

Parámetros de Configuración:

  • URL Base: Dirección completa del servidor Flowise
  • API Key: Clave de autenticación (opcional según configuración)
  • Flow ID: Identificador específico del flujo de validación
  • Timeout: Tiempo máximo de espera en segundos
  • Habilitación: Switch para activar/desactivar procesamiento IA

Funcionalidades Avanzadas:

  • Test de conexión: Verificación en tiempo real con métricas
  • Métricas de respuesta: Tiempo de procesamiento y estadísticas
  • Reintentos automáticos: Manejo inteligente de errores de conexión
  • Monitoreo de salud: Dashboard de estado del servicio IA

💼 Casos de Uso Comunes

Caso 1: Onboarding de Nuevo Transportista

  1. Registrar chofer en módulo Choferes con datos completos
  2. Registrar vehículos en módulos Camiones/Acoplados
  3. Crear equipo en módulo Equipos con asociación inteligente
  4. Asignar cliente según requisitos específicos
  5. Cargar documentos vía portal móvil o carga masiva
  6. Aprobar documentos pendientes con validación manual

Caso 2: Intercambio de Vehículos (Swap)

  1. Acceder a Equipos y localizar equipo origen
  2. Seleccionar equipo que requiere modificación
  3. Gestionar componentes → Cambiar camión/acoplado específico
  4. Confirmar operación con auto-cierre del equipo anterior
  5. Validar nueva configuración y estado documental

Caso 3: Auditoría Masiva por Cliente

  1. Ir a Consulta y preparar criterios de búsqueda
  2. Cargar CSV con DNIs a verificar del cliente
  3. Revisar resultados con semáforos por equipo
  4. Descargar ZIP masivo con documentación vigente
  5. Generar reporte de cumplimiento consolidado

🔧 Troubleshooting

Problemas Comunes y Soluciones

⚠️ Documentos no se clasifican automáticamente

  • Verificar conexión Flowise en /configuracion/flowise
  • Confirmar que el Flow ID es correcto y activo
  • Revisar logs de procesamiento en Dashboard
  • Validar timeout y configuración de red

🔍 Búsquedas sin resultados

  • Verificar formatos de DNI (solo números, sin puntos)
  • Confirmar normalización de patentes (mayúsculas, sin espacios)
  • Revisar filtros por dador de carga activo
  • Validar que los equipos estén en estado activo

⚡ Problemas de Performance

  • Usar paginación en listas grandes (>100 elementos)
  • Aplicar filtros específicos en búsquedas amplias
  • Revisar estado de jobs de procesamiento en background
  • Monitorear métricas de sistema en health check

Contacto y Soporte

Para problemas técnicos o consultas sobre funcionalidades específicas, consultar con el equipo de desarrollo o revisar los logs del sistema en el módulo de health check disponible en /health/detailed.
